# MERNShop — Full Stack MERN Application

A complete production-ready e-commerce application built with:

- **Frontend**: Next.js 15, TypeScript, Tailwind CSS, Zustand, Socket.IO
- **Backend**: Node.js, Express 4, TypeScript, MongoDB/Mongoose, JWT, Socket.IO
- **Database**: MongoDB Atlas (free tier)
- **Image storage**: Cloudinary (free tier)

## Quick Start (Local Development)

### Prerequisites
- Node.js 20+
- MongoDB Atlas account (free) — https://cloud.mongodb.com
- Cloudinary account (free) — https://cloudinary.com

### 1. Backend Setup

```bash
cd backend

# Install dependencies
npm install

# Configure environment
cp .env.example .env
# Edit .env — fill in MONGODB_URI, JWT_SECRET, JWT_REFRESH_SECRET
# (Cloudinary and SMTP are optional for basic testing)

# Generate products data file
npm run seed

# Insert products into MongoDB
npm run db:seed

# Create admin user (admin@mernshop.com / Admin@1234)
npm run db:admin

# Start development server
npm run dev
# → API running at http://localhost:5000
# → Swagger docs at http://localhost:5000/api/docs
```

### 2. Frontend Setup

```bash
cd frontend

# Install dependencies
npm install

# Configure environment
cp .env.local.example .env.local
# The defaults work if backend is on localhost:5000

# Start development server
npm run dev
# → App running at http://localhost:3000
```

### 3. Open the app

| URL | Description |
|-----|-------------|
| http://localhost:3000 | Main shop |
| http://localhost:3000/auth/login | Customer login |
| http://localhost:3000/admin/login | Admin dashboard |
| http://localhost:5000/api/health | Backend health check |
| http://localhost:5000/api/docs | Swagger API docs |

**Admin credentials**: `admin@mernshop.com` / `Admin@1234`

## Environment Variables

### Backend (`backend/.env`)

| Variable | Required | Description |
|----------|----------|-------------|
| `PORT` | No | Server port (default: 5000) |
| `NODE_ENV` | No | `development` or `production` |
| `MONGODB_URI` | **Yes** | MongoDB Atlas connection string |
| `JWT_SECRET` | **Yes** | 32+ char random string |
| `JWT_REFRESH_SECRET` | **Yes** | Another 32+ char random string |
| `FRONTEND_URL` | No | Frontend URL for CORS (default: http://localhost:3000) |
| `CLOUDINARY_CLOUD_NAME` | No* | Required for image uploads |
| `CLOUDINARY_API_KEY` | No* | Required for image uploads |
| `CLOUDINARY_API_SECRET` | No* | Required for image uploads |
| `SMTP_HOST` | No* | Required for emails |
| `SMTP_USER` | No* | Required for emails |
| `SMTP_PASS` | No* | Required for emails |

Generate JWT secrets:
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### Frontend (`frontend/.env.local`)

| Variable | Required | Description |
|----------|----------|-------------|
| `NEXT_PUBLIC_API_URL` | No | Backend URL (default: http://localhost:5000) |
| `NEXT_PUBLIC_BASE_URL` | No | Frontend URL (default: http://localhost:3000) |

## API Endpoints

### Auth
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/api/v1/auth/register` | — | Register customer |
| POST | `/api/v1/auth/login` | — | Login |
| POST | `/api/v1/auth/logout` | — | Logout |
| POST | `/api/v1/auth/refresh` | Cookie | Refresh access token |
| GET | `/api/v1/auth/me` | Bearer | Get current user |
| POST | `/api/v1/auth/forgot-password` | — | Send reset email |
| POST | `/api/v1/auth/reset-password` | — | Reset password |

### Products
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| GET | `/api/v1/products` | — | List products (filter, sort, paginate, search) |
| GET | `/api/v1/products/:id` | — | Get product by ID |
| GET | `/api/v1/products/stats` | — | Category stats |
| POST | `/api/v1/products` | Admin | Create product |
| PUT | `/api/v1/products/:id` | Admin | Update product |
| DELETE | `/api/v1/products/:id` | Admin | Delete product |
| POST | `/api/v1/products/:id/image` | Admin | Upload image to Cloudinary |
| GET | `/api/v1/products/:id/reviews` | — | Get reviews |
| POST | `/api/v1/products/:id/reviews` | Bearer | Submit review |
| DELETE | `/api/v1/products/:id/reviews/:reviewId` | Bearer | Delete own review |

### Orders
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/api/v1/orders` | Bearer | Place order (atomic transaction) |
| GET | `/api/v1/orders/my` | Bearer | My orders |
| GET | `/api/v1/orders` | Admin | All orders |
| PUT | `/api/v1/orders/:id/status` | Admin | Update order status |

## Deployment

### Backend → Render

1. Create account at [render.app](https://render.app)
2. New Project → Deploy from GitHub → select repo
3. Set **Root Directory**: `backend`
4. Add **MongoDB** database from Render's plugin marketplace
5. Set environment variables in Render dashboard
6. After deploy, open Render Shell and run:
   ```bash
   npm run db:seed
   npm run db:admin
   ```

### Frontend → Vercel

1. Push code to GitHub
2. Import repo at [vercel.com](https://vercel.com)
3. Set **Root Directory**: `frontend`
4. Add environment variable:
   ```
   NEXT_PUBLIC_API_URL=https://your-backend.up.render.app
   ```
5. Deploy

### Update CORS after deployment

In Render, update `FRONTEND_URL` to your Vercel URL:
```
FRONTEND_URL=https://your-app.vercel.app
```
